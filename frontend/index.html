STEP 1: THE PROBLEM (UNSALTED)

An unsalted hash is just the output of a hash function (like SHA-256). If two users have the same password, they have the same hash. An attacker can pre-compute hashes for common passwords (a "Rainbow Table") and instantly crack them.

STEP 2: THE SOLUTION (SALTING)

      
A "salt" is a unique, random string added to *each* password before hashing. Now, two identical passwords will have different hashes. This makes rainbow tables useless.

STEP 3: THE MODERN STANDARD (BCRYPT)

      
Modern functions like Bcrypt and Argon2 automatically generate unique salts and include them *inside* the final hash string. They are also intentionally "slow" (key stretching) to resist brute-force attacks.
